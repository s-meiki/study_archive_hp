# Legal Compliance Guide

調査日: 2026-03-10

このメモは、現在の実装を前提にした運用整理です。対象は次の機能です。

- `public/` の静的ページを Next.js から配信する公開サイト
- `admin/` でローカル編集し、`public/data/` と `public/uploads/` を更新する運用
- YouTube URL の掲載と、詳細ページでの遅延埋め込み
- `/contact` の問い合わせフォーム
- PDF などの資料アップロード

これは実務メモであり、最終的な法的判断そのものではありません。病院、薬局、法人の実名を出して運用する場合や、患者情報を扱う場合は、公開前に法務又は管理部門の確認を入れる前提で使ってください。

## 1. 今のプロダクトで特に重要な法律

### 1-1. 個人情報保護法

ひとことで言うと:
人が誰か分かる情報を、勝手に集めたり、広めたり、ずさんに扱ったりしてはいけないというルールです。

このサイトで関係する理由:

- 登壇者名、所属、写真、連絡先などを載せる可能性がある
- 資料や動画に患者情報、施設名、症例番号、検査日などが残るおそれがある
- 医療情報は特に慎重に扱うべき情報として扱われる

中学生向けの対策:

- 名前を出す必要がある人だけを載せる
- 患者さんの名前、顔、ID、病院番号、生年月日、細かすぎる日付は消す
- 「この人の情報を何のために載せるか」を最初に決める
- いらない情報は最初から載せない
- 間違って公開したら、まず公開停止、次に関係者への連絡、その後に報告手順を動かす

運用で必ずやること:

- 公開前に `患者情報チェック` を通す
- 登壇者の氏名や所属を出すときは、公開許可を残す
- プライバシーポリシーを公開する
- 問い合わせ窓口を決める
- アクセスできる編集者を絞る
- 漏えい時の初動手順を 1 枚にまとめる

漏えい時の実務メモ:

- 要配慮個人情報や大きな漏えいは、個人情報保護委員会への報告と本人通知が必要です
- 個人情報保護委員会の公開資料では、速報は把握からおおむね 3 から 5 日以内、確報は原則 30 日以内、不正目的のおそれがある場合は 60 日以内の整理です

### 1-2. 医療・介護分野の個人情報ガイダンス

ひとことで言うと:
医療関係の個人情報は、ふつうの個人情報よりも、もっと慎重に扱ってくださいという実務ルールです。

このサイトで関係する理由:

- 勉強会の資料や録画は、匿名化が甘いと患者情報がそのまま残りやすい
- 症例検討の材料は、参加者や公開範囲で扱いが変わる

中学生向けの対策:

- 「名前を消したから大丈夫」と思わない
- 珍しい病気、年齢、時期、病院名がセットになると、誰か分かることがある
- 外部の人に見せる資料は、内部用資料より厳しく削る

運用で必ずやること:

- 症例入り資料は `公開版` を別で作る
- 原本 PDF をそのまま上げない
- 匿名化と公開可否の確認者を決める

### 1-3. 電気通信事業法の外部送信規律

ひとことで言うと:
サイトを開いたり再生したりしたときに、利用者の端末情報などが外部サービスへ送られるなら、そのことを分かりやすく知らせるルールです。

このサイトで関係する理由:

- 詳細ページで YouTube 埋め込みを使う
- 現在の実装は `youtube-nocookie.com` を使い、しかもボタンを押した時だけ読み込む
- それでも、再生時には YouTube / Google 側へ情報が送られることがある

中学生向けの対策:

- 何が送られるかを先に書く
- どこに送られるかを書く
- 何のためかを書く
- 難しい英語リンクだけ置いて終わりにしない

運用で必ずやること:

- プライバシーポリシー又は外部送信通知に、日本語で `送信される情報の内容` `送信先の名前` `利用目的` を載せる
- 動画は自動再生しない
- `再生する` を押した時だけ iframe を読み込む
- 動画ブロックの近くに、YouTube 読み込みの注意文とプライバシーポリシーへのリンクを置く

### 1-4. 問い合わせフォームで扱う個人情報

ひとことで言うと:
問い合わせフォームで名前やメールアドレスを受け取るなら、その情報の使い道と送信先をきちんと説明しなければいけません。

このサイトで関係する理由:

- `/contact` で氏名、メールアドレス、問い合わせ内容を受け取る
- Cloudflare Turnstile でボット判定をする
- 送信内容を Discord Webhook へ通知する

中学生向けの対策:

- 何を入力してもらうかを最小限にする
- どこへ送られるかを隠さない
- 患者情報や未匿名化症例情報は送らないように書く

運用で必ずやること:

- フォームの近くに `Cloudflare Turnstile` と `Discord` 利用を明示する
- プライバシーポリシーで送信先、送信情報、利用目的を書く
- 受け取った問い合わせを長く放置しない

### 1-5. 著作権法

ひとことで言うと:
他人が作った動画、スライド、論文図表、PDF、画像、ロゴを、勝手に載せたり配ったりしてはいけないというルールです。

このサイトで関係する理由:

- スライドや参考資料 PDF を `public/uploads/` に置ける
- ガイドライン、論文、他院資料、学会資料をそのまま再配布すると危ない
- OGP 画像やサムネイルも権利確認が必要

中学生向けの対策:

- 自分たちが作った資料だけを基本にする
- 他人の資料は、許可がある時だけアップする
- 許可がない時は `ファイルを再配布せず、元ページへのリンクだけ` にする
- 引用は、必要最小限で、出典を書き、引用が主役にならない形で使う

運用で必ずやること:

- すべてのアップロード資料に `権利の根拠` を残す
- `owned`, `licensed`, `link-only`, `quotation` のどれかで管理する
- 出典 URL と許可日を残す
- 権利不明の PDF は公開しない

### 1-6. 医療法の広告規制

条件:
病院、診療所、薬局などの公式サイトの一部として公開し、一般向けの広告として評価されうる場合に特に重要です。

ひとことで言うと:
病院や診療所のサイトが広告として見られるときは、うそ、大げさ、比較優良、体験談まかせの表現が禁止されるルールです。

このサイトで関係する理由:

- このサイト自体は教育アーカイブですが、病院や診療所の公式ページの一部として出すと、広告規制の話に近づきます
- 治療成績の誇張や、患者体験談のような見せ方をすると危険です

中学生向けの対策:

- 「日本一」「必ず良くなる」「安全です」と言い切らない
- 教育目的のアーカイブと、集客のための医療広告を混ぜない
- 患者さんの声を宣伝材料にしない

運用で必ずやること:

- 画面上に `教育用コンテンツ` と明示する
- 個別治療の推奨と読まれる書き方を避ける
- 医療機関の宣伝ページと同じ導線にする場合は、別途レビューする

### 1-7. 薬機法の広告規制

条件:
一般向け画面で、特定の医薬品や医療機器の効能効果を宣伝する見え方になる場合に特に重要です。

ひとことで言うと:
薬や医療機器について、効き目を大げさに言ったり、誤解させる宣伝をしてはいけないというルールです。

このサイトで関係する理由:

- 勉強会タイトルや要約に、特定の薬や治療法が出ることがある
- 一般向けに見える形で、効能を断定したり、未承認や適応外の内容を宣伝調で見せると危ない

中学生向けの対策:

- 「この薬は絶対効く」と書かない
- 登壇内容の要約を、宣伝文ではなく学習メモとして書く
- 未承認、適応外、施設独自運用は、必要ならその旨をはっきり書く

運用で必ずやること:

- 薬剤名や製品名がある要約は、宣伝文調にしない
- 効果保証、体験談保証、比較優良表現を避ける
- 一般向けページで販売誘導と組み合わせない

## 2. 関連する利用規約・ポリシー

### 2-1. Vercel Terms / Privacy Policy

関係する理由:

- このサイトは Next.js アプリとして Vercel 等のホスティング基盤で配信される場合がある

気をつけること:

- 秘密情報や個人情報を公開コードや公開レスポンスへ載せない
- ホスティング事業者が処理するログや通信情報をプライバシーポリシーで説明する

### 2-2. Cloudflare Turnstile Terms / Privacy Policy

関係する理由:

- 問い合わせフォームでボット判定に Turnstile を使う

気をつけること:

- フォーム画面で外部サービス利用を明示する
- Cloudflare へ送信される情報と目的を説明する

### 2-3. Discord Terms / Privacy Policy

関係する理由:

- 問い合わせ内容を Discord Webhook に送る

気をつけること:

- 問い合わせ内容が Discord に送られることを説明する
- 患者情報や不要な個人情報を送らない運用にする

### 2-4. YouTube Terms of Service / Embedded Player rules / Google Privacy Policy

関係する理由:

- 動画再生に YouTube 埋め込みを使う
- `youtube-nocookie.com` を使っても、完全に何も送られないわけではない

気をつけること:

- 動画は権限のある YouTube URL だけを埋め込む
- 埋め込み前に注意文を出す
- 外部サービスの規約とプライバシーポリシーへの案内を用意する

## 3. 設計図に必ず入れるべきもの

今のプロダクトで必須なのは次の項目です。

- フッター又は常設領域に `利用規約`, `プライバシーポリシー`, `問い合わせ先` のリンク
- 一覧ページと詳細ページの両方に `教育用コンテンツ` の明示
- YouTube 読み込み前の注意文
- 公開前チェック: `患者情報`, `登壇者公開許可`, `著作権`, `外部リンク先`
- 権利侵害や削除依頼を受けるための窓口
- 誤掲載時の削除・差し替え手順

## 4. 公開前チェックリスト

- 患者名、病院 ID、顔写真、細かすぎる日付が消えている
- スピーカー名、所属、写真の掲載許可が確認できている
- PDF やスライドの権利根拠が残っている
- YouTube URL が埋め込み可能か確認した
- 利用規約とプライバシーポリシーのリンクが表示されている
- `教育目的であり、個別診療の代替ではない` と表示されている
- 外部送信の説明が日本語で読める
